# Learning GitHub
It's self-learning at a beginner level, the first time using GitHub thru the GitHub Desktop GUI, which helps users manage projects and files easily and understandably.

## Lessons Learned
1. "Repository" or "REPO"  creation on Web [browser](https://github.com/KartoonRMUTTO)
2. "Clone" a repository : Cilck “File” > “Clone repository” choose Repo from GitHub browser choose Local Path > click clone
3. "Commit" is the creation of a save point on the computer, and it is necessary to specify a Description.
4. "Push origin" to send the changes to the repository on the GitHub browser.
5. "File" Creating files in the folder that we "Clone" to the computer by opening the folder in the IDE VS Code can be added, edited, and deleted. 
6. "Revert" It's a recall.  click History > Right-click > Revert change in commit
7. "Branch" Can develop code or modify the project in a separate section without affecting the main
Current branch (In the middle) > click "New Branch" > Name the new branch to want > click "Create Branch"
8. "Merge" Merging the changes made in the "Branch" back into the "main"
Current branch (In the middle) > click Main > Choose a branch to merge into main > choose Branch to want > 
Create a merge commit > click "Push origin"
9. "Pull Request" For team collaboration, you can view the details of changes, comment, and review the code.
